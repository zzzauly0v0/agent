# 三种文本分类方法对比汇总

## 一、总体对比

| 对比维度 | BERT Fine-tune | LLM Zero-shot | LLM SFT（LoRA） |
|---------|---------------|---------------|------------------|
| 模型 | bert-base-chinese（110M） | Qwen2-0.5B（500M） | Qwen2-0.5B（500M） |
| 核心原理 | 预训练 + 微调，在分类头学习类别边界 | 利用预训练知识理解指令，不更新参数 | 冻结原模型，训练低秩旁路矩阵适配任务 |
| 核心技术 | CLS/Mean/Max 池化（样本不均衡加权 loss） | Prompt Engineering | LoRA + Loss Masking |
| 可训练参数 | 102.3M（全量） | 0 | 1.08M（0.22%，LoRA r=8） |
| 训练成本 | 1 epoch，60min+/epoch（CPU） | 无需训练 | 1 epoch，约 60+ min（CPU） |
| 推理速度（CPU） | 0.02–0.05 秒/条 | 0.69 秒/条 | 0.56 秒/条 |
| 实测准确率 | 52%~55%（10000条） | 36%（200条） | **55%**（val 200条，5K训练） |
| 无法解析率 | 不存在 | 29%（200条） | 1.5%（200条） |
| 数据需求 | 需标注数据（全量） | 无需标注 | 需部分标注数据 |
| 修改成本 | 需重新训练 | 改 prompt 即可 | 需重新微调 |
| 适用场景 | 有大量标注数据、追求极致精度 | 快速原型、零标注 | 少量标注 + 大模型能力 |

> **结论：针对该项目文本分类，建议使用 LLM SFT（LoRA）方案。**

---

## 二、说明 1：BERT Fine-tune 中 CLS / Mean / Max 三种池化策略对比

| 维度 | CLS + 普通 CrossEntropyLoss | Mean + 普通 CrossEntropyLoss | Max + 加权 CrossEntropyLoss |
|------|------------------------------|--------------------------------|--------------------------------|
| 特点 | 标准 BERT 分类，利用 [CLS] 语义 | 利用全序列信息，更鲁棒 | 关注最强特征 + 处理类别不平衡 |
| 适用场景 | 类别均衡 + 短文本 | 类别均衡 + 长文本 | 类别严重不平衡 |
| 最优 acc | 0.5570 | 0.5335 | 0.5100 |
| Train 样本数 | 53360 条 | 10000 条 | 10000 条 |
| Val 样本数 | 10000 条 | 2000 条 | 2000 条 |
| Test 样本数 | 10000 条 | 2000 条 | 2000 条 |
| 准确率 | 5569/10000 = 0.5569 | 5492/10000 = 0.5492 | 5302/10000 = 0.5302 |

### 总结
- 加权 CrossEntropyLoss 可以显著缓解类别不平衡问题，将样本数偏少类别的 F1-score 提升。
- CLS + 普通 CrossEntropyLoss 的 acc 最优，可能与该策略训练样本数（53360条）远多于其他两种策略（10000条）有关。
